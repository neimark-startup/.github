
## ❌ Почему не стоит коммитить базу данных:

1. **Бинарные файлы** не отслеживаются Git эффективно
2. **Размер репозитория** будет постоянно расти
3. **Конфликты слияния** невозможны для бинарных файлов
4. **Конфиденциальные данные** могут попасть в репозиторий
5. **Разные среды** (разработка/продакшн) требуют разных данных

## ✅ Правильное решение:

### 1. Добавьте файл БД в `.gitignore`:
```gitignore
# Database
*.sqlite3
*.db
db.sqlite3
```

### 2. Используйте миграции Django:
```bash
# Создайте миграции
python manage.py makemigrations

# Примените миграции
python manage.py migrate
```

### 3. Для начальных данных используйте фикстуры:
```bash
# Создать фикстуру
python manage.py dumpdata <app_name> > fixtures/initial_data.json

# Загрузить фикстуру
python manage.py loaddata initial_data.json
```

### 4. Если база уже в репозитории:
```bash
# Удалить из индекса Git
git rm --cached db.sqlite3

# Добавить в .gitignore
echo "db.sqlite3" >> .gitignore

# Закоммитить изменения
git add .gitignore
git commit -m "Remove database file and add to gitignore"
```

Это обеспечит чистоту репозитория и правильную работу команды разработчиков.
